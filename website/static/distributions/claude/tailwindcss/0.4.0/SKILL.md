# Tailwind CSS

Use this skill when the task depends on Tailwind CSS-specific APIs, workflows, or debugging guidance rather than generic library advice.

## Purpose

This skill teaches an agent to use Tailwind CSS 4.2.1 for CSS-first styling, theme customization, utilities, variants, directives, compatibility constraints, and migration work without collapsing Tailwind-specific behavior into generic CSS or framework advice.

## When to use this skill

- Tailwind CSS setup and implementation work
- Tailwind CSS API usage and configuration decisions
- Tailwind CSS-specific debugging and maintenance
- version-sensitive framework or tool guidance

## Working style

- Prefer the v4 CSS-first workflow over stale config-heavy habits when working in current Tailwind codebases.
- Distinguish Tailwind directives, theme variables, utility classes, and ordinary CSS rules instead of mixing them into one vague styling model.
- Treat `@source` and class detection as build-time concerns, not runtime CSS behavior.
